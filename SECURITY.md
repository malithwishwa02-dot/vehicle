# Security & Vulnerability Disclosure

We take security seriously. If you discover a security vulnerability affecting this repository, please disclose it responsibly.

Steps to report:

1. Open a private issue titled `SECURITY:` and include a clear, concise description of the vulnerability and steps to reproduce.
2. Add relevant artifacts (logs, small repro) but avoid attaching secrets or full profiles.
3. If you prefer, request contact details in the issue to exchange private information.

We will respond within 72 hours and coordinate fixes and disclosure.

Note: This project contains sensitive tooling and test data. Do **not** commit secrets, full browser profiles or production data to the repository. Use `.gitignore` to exclude generated artifacts such as `generated_profiles/` and local virtual environments.
